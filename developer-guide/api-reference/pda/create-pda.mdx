You have to write a mutation to create a PDA. The mutation should be written in the following format.

You need to fill up the parameters according to the data model you are using.

<Check>
  If you are issuing the PDA on behalf of an organization, you need to provide
  the organization's `GATEWAY_ID` as the value of the `organization` parameter.
</Check>

| Parameter      | Description                                                                                      | Possible Values | Possible Types                                |
| -------------- | ------------------------------------------------------------------------------------------------ | --------------- | --------------------------------------------- |
| title          | The title of the PDA.                                                                            | String          |                                               |
| description    | A brief description of the PDA.                                                                  | String          |                                               |
| owner          | The person you are issuing the PDA to.                                                           | Object          | GATEWAY_ID, EMAIL, EVM, SOLANA, POKT, USER_ID |
| dataModelId    | The ID of the data model you are using.                                                          | String          |                                               |
| expirationDate | The format is ISO 8601. If null, the PDA will not expire.                                        | String          |                                               |
| organization   | The organization who is issuing the PDA. Can be null.                                            | Object          | GATEWAY_ID, ORG_ID                            |
| claim          | The fields which are there in the data model structure. You need to fill up the required fields. | Object          |                                               |

```graphql
# Query
mutation createPDA {
  createPDA(
    input: {
      title: "My Personal Data Asset"
      description: "This is an example PDA showcasing filler data."
      owner: { type: GATEWAY_ID, value: "user-gateway-id" }
      dataModelId: "example-data-model-id"
      expirationDate: null
      organization: { type: GATEWAY_ID, value: "org-gateway-id" }
      claim: {
        name: "John Doe"
        age: 30
        email: "johndoe@example.com"
        interests: ["reading", "traveling", "photography"]
        isStudent: false
      }
    }
  ) {
    id
    arweaveUrl
    dataAsset {
      owner {
        id
        gatewayId
      }
      issuer {
        id
        gatewayId
      }
    }
    transactionId
  }
}
```

You have to put in the parameters according to the data model you are using.

The `dataModelId` is the ID of the data model you are using.

You can fetch an already prepared query from the dashboard for the data model you are using.
